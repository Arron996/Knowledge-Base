# 退票可退维度问题集

> applyRefund 迁移期沉淀的可退逻辑与边界场景。2024 H1 搭建。

---

## 处理流程（新票台）

```
build context（主单/子单/乘客/segment/ticket）
  → check 幂等（订单状态）
  → 异步调 VC（按子单）
  → handle response
      → refundable / partialRefundable
      → 单票 RefundTicket / 多票 commonRefundTicket / 整单 allRefundTicket
```

契约：`Request → InnerRequest → SupplierApplyRefundRequest`

---

## 维度映射

| 层级 | 关系 |
|------|------|
| orderId | → n 个 orderItemId |
| orderItemId | → n 个 segmentId / ticketId |
| uktis | 退票维度：去程 / 返程 / 去返程（leg id） |
| 英铁 mticket | 按行程+乘客+站台+时间唯一确定 ticket；自拆单**不能分段退，只能全退** |

### 乘客两层

- **gdsPassengers**：订单层（如 4 大人 1 婴儿）
- **SupplierPassengers**：供应层，仅出票乘客

---

## 典型难题

### 1. P2P + X 绑定退

- 供应返回 P2P&X 不可整单退，但 P2P 可整退
- `refundableItem.productServiceType` 随单产品；`refundMode` 单退 / 随单退
- 供应标 partial must，但多段 refundableItem 可**拼接成整单退**

示例：子单 A→B→C，乘客 ①②③，整单 partial must，但 ①+③ 两段可组合退。

### 2. 车次延误

特殊 mode / 时间窗参数触发可退（与供应 refundFlag 联动）。

### 3. 时间规则

- 出票 **20 分钟内**退票免手续费
- **advance ticket** 20 分钟外不可退（VC 错误码如 `620031`）

### 4. VC 返回形态

| refundFlag | 含义 |
|------------|------|
| REFUND_ALL_OR_PARTIAL | 可整可部 |
| REFUND_PARTIAL_MUST | 必须部分退 |
| NO_REFUND | 不可退 |

聚合逻辑：子单维度可退/不可退 → 行程段 → 票维度 `ticketList` / `commonTicketList`。
