## 一、BSC 员工汇总 · 综合评价

> 填 workDay **BSC 页最底部「员工汇总 → 综合评价」**（必填）。

上半年工作围绕票台出退改核心链路与系统治理展开，整体达成度较好。

**业务与方案方面**，O1 六条 KR 全部完成，包括 X 产品扩展、delayRepay、车次停运专项、售后变更 BI、转人工工单、SNCFR 选座等；在 applyRefund 改造、15560 下线、Admin 退改切 VC 等项目中，方案侧重系统性治理与风险可控的灰度迁移，而非单点修补。

综合来看，上半年重点任务均已闭环，架构治理与业务交付达到预期，稳定性与排障效率也有可感知提升；Agent 与工具链仍在持续摸索，下半年会在交付质量与个人效能上继续加强。

## 八、领导力

> 填 workDay **领导力** 页：每项评分 **4-经常**，并粘贴对应「领导力评语」；底部填「领导力总结评语」。

### 1. 独立思考，洞察本质（思考力）｜4-经常

**领导力评语：**

在 applyRefund 契约改造中，没有停留在局部修 bug，而是从老链路结构出发设计隔离层、灰度开关与飞书异步对比，把「可迁移、可回滚、可观测」作为上线前提，降低大规模改造对线上的冲击。

面对婴儿退、异常退等复杂场景，能从 VC 规则与票台处理链路整体考虑，抽象成人/婴儿绑定等更通用的处理方式，兼顾当前问题闭环与后续特殊乘客类型的扩展空间。

在监控建设上，不满足于人工翻日志或看板，主动用飞书 Agent 把流量、对比、异常信息推到面前，把「人找问题」变成「问题找人」，体现对运维痛点的独立判断。

---

### 2. 主动行动，结果导向（执行力）｜4-经常

**领导力评语：**

主动推进三条架构主线（applyRefund 全量、15560 完全下线、Admin 退改切 VC），长周期任务按阶段拆解、按节奏灰度，H1 内均达成全量或下线目标，没有因改造周期长而搁置收尾。

对卡单、异常退锁、TIS 压测等风险点主动发起治理：卡单监控多轮优化并上线；TIS 在测试环境压测联调，推动相关方修复；后续供应侧线上波动时，因异常配置已前置，问题未进一步扩大。

在跨团队事项上（如 TIS 联调、Admin 与票台对齐、灰度期问题跟进）能持续推动闭环，沟通聚焦现象、日志与结论，减少反复空转。

---

### 3. 客户导向，学以致用（自驱力）｜4-经常

**领导力评语：**

从一线排障和人工处理体验出发，优化 Admin 出退列表、日志跳转、异常配置审批等能力，并通过页面联动、排查小组件减少多系统来回切换，提升异常单处理效率（属运维侧效能提升，P3 量级）。

持续学习 Cursor、Agent、MCP 等工具，将 Admin 票台日志 MCP 融入日常定位问题流程，并个人沉淀简化排障与联调步骤的技巧，缩短「描述问题 → 拿日志 → 定根因」的路径。

对 Agent 辅助研发保持自驱：已形成一定工作习惯，但仍在摸索阶段，尚未形成团队级推广，后续可在稳定个人提效的基础上逐步沉淀可复用方法。
