# 票台排障路径

> 日常联调与线上问题定位的默认链路。

---

## 标准路径

```
GDS 票台 / 新票台服务
  → Offline（人工操作页面）
  → Admin（订单/票台日志）
  → BAT CLOG / Trace
```

2024 笔记中的 shorthand：`gds → offline → admin → bat`

---

## 按场景

| 场景            | 建议入口                                                                |
| ------------- | ------------------------------------------------------------------- |
| VCC 实付        | CLOG `appid=100051894`，`%vcc end%`；见 [[领域/VCC实付对账模型]]               |
| 人工出票          | Offline 列表 → 查 operator / 状态 → Admin ticket log                     |
| applyRefund   | Admin 退票日志 + VC 返回 refundFlag                                       |
| createInvoice | `supplier='iryos' and invoice_status in(-1,0) and order_status=600` |
| obbat 取票      | `obtain_status`；job 链：update → queryVc → upload → send              |
| 新票台列表         | 查询时间限制 `< 3 days`（OrderItemVOList）                                  |

---

## Offline 日志（新票台）

| 问题 | 结论 |
|------|------|
| 日志放哪层 | infrastructure 状态机取不到参数 → **biz 层** |

| 失败信息 | message（是否要带 origin message 视场景） |
